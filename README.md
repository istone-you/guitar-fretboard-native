# Guitar Fretboard - ギター学習アプリ

指板上に音名・度数・スケール・コードを重ね合わせて表示するギター学習ツール。

## 機能一覧

### 指板表示

- 6弦×15フレット（0〜14フレット）
- 音名表示 / 度数表示の切り替え
- ルート音の選択（全12音）
- フレット範囲の指定
- ダブルタップで横画面表示
- ポジションマーク（3, 5, 7, 9, 12フレット）

### レイヤーシステム

最大3つのレイヤーを重ねて表示。各レイヤーは独立した色を持ち、左のチェックボックスで表示/非表示を切り替え可能。iOS 26スタイルのインタラクションを採用:

- **タップ**: レイヤー行をタップして編集モーダルを開く
- **長押し**: iOS 26ネイティブ風コンテキストメニューを表示（編集・複製・削除）
- **左スワイプ**: 削除アクション
- **ドラッグ&ドロップ**: 行全体を長押し（200ms）してから縦にドラッグして並べ替え。シフト移動方式（間の要素も詰めて移動）。空スロットはドロップ先として有効だがドラッグ不可。`react-native-reanimated-dnd` を使用し、`layer.id` をキーとすることでドロップ後のちらつきを防止

同じセルに複数レイヤーが重なる場合は、同心円状に入れ子でドットが表示される（外側が1番目、内側が後のレイヤー）。

レイヤー設定モーダルの右上「?」ボタンから、レイヤータイプの解説と選択中のスケール・コードの解説を確認できる。レイヤータイプの説明は常に表示され、スケールやコードを切り替えると個別の解説のみが動的に切り替わる。

#### スケールレイヤー

対応スケール:
- 基本: メジャー、ナチュラルマイナー
- ペンタトニック: メジャーペンタ、マイナーペンタ
- ブルーノートスケール
- マイナー派生: ハーモニックマイナー、メロディックマイナー
- モード: Ionian, Dorian, Phrygian, Lydian, Mixolydian, Aeolian, Locrian
- 応用: フリジアンドミナント、リディアンドミナント、オルタード、ホールトーン、ディミニッシュ

#### コードレイヤー

表示形式:
- **コードフォーム**: 6弦/5弦ルートのバレーコード + オープンコードを枠付きで表示
  - 対応: Major, Minor, 5, 7th, maj7, m7, m7(b5), dim7, m(maj7), sus2, sus4, 6, m6, dim, aug, 9, b9, #9, add9, 11, #11, add11, add#11, 13, b13
  - コード枠の表示/非表示切り替え
- **トライアド**: 3弦セット（1-3弦, 2-4弦, 3-5弦, 4-6弦）×転回形（基本/第一/第二）
  - 対応: Major, Minor, Diminished, Augmented
- **ダイアトニック**: キー（メジャー/マイナー）×和音（3和音/4和音）から度数を選択
- **オンコード**: スラッシュコード（C/E, Am/G 等）のボイシングを表示
  - ルート音に応じた一覧から選択
  - 200以上のオンコードに対応

#### CAGEDレイヤー

- C・A・G・E・Dの5フォームを複数選択で表示
- メジャー / マイナーの切り替え
- コード枠の表示/非表示切り替え

#### カスタムレイヤー

- 音名または度数を自由に選択して指板上に表示
- 度数は拡張テンション対応（♭9, 9, ♯9, 11, ♯11, ♭13, 13）
- コードの構成音を抽出する機能（コードタイプを選択して抽出）
- リセット機能

**応用例:**
- コードタイプから構成音を抽出し、コードトーンの指板上の配置を把握
- 特定の度数だけを選んで、指板上のインターバルパターンを視覚化
- オクターブの位置関係を把握（同じ音名だけを選択して配置パターンを確認）
- スケールレイヤーと重ねて、コードトーンとスケール音の関係を視覚的に比較

#### プログレッションレイヤー

コード進行を"表示するため"ではなく、**他レイヤーを固定したままコードだけを順に差し替えるため**のレイヤー。

- **進行テンプレート**: I-IV-V / ii-V-I / I-vi-ii-V / I-VI-IV-V / I-V-vi-IV / iii-VI-ii-V / I-III-IV-iv / minor ii-V-i / i-VII-VI-VII / 12bar blues（固定）
- **キータイプ**: Major / Minor を選択（キールートはヘッダーのルート音を使用）
- **コードサイズ**: 組み込みテンプレートは3和音固定。カスタムテンプレートはコードごとに個別指定
- **ステップ操作**: レイヤーパネルの ‹ / › ボタンで前後のコードへ即時移動
- **ゴースト表示**: 前後ステップのコードを同色の低透明度で重ねて表示（任意ON/OFF）
- 他レイヤー（スケール / CAGED / カスタム）と自由に組み合わせ可能

**活用例:**
- Progression + Scale: コード進行を辿りながら各コードでスケールの使われ方を比較
- Progression + CAGED: 進行の各コードをCAGEDフォームで把握
- Progression + Custom（3rd/7th）: ガイドトーンがどのコードでどこに現れるかを確認

#### プリセット

- 現在のレイヤー構成に名前を付けて保存
- 保存したプリセットをワンタップで読み込み
- プリセットの削除は管理タブで行う

### 管理タブ

フッター4番目のタブ（アイコン: tray.full）。レイヤープリセットとカスタム進行テンプレートを一元管理する。

#### レイヤープリセット

- 保存済みプリセットを一覧表示（名前・レイヤー数）
- タップで詳細ボトムシートを表示（各レイヤーのカラードット・タイプ・内容を読み取り専用で確認）
- 左スワイプで削除（赤背景＋ゴミ箱アイコン）
- 長押し200ms → ドラッグで並べ替え（右端ハンドル）

#### コード進行テンプレート

- ユーザー作成テンプレートのみ表示（ビルトインは非表示）
- ＋ボタン → 作成ボトムシート（名前入力・音度セレクター + コードタイプ選択でコードを追加、3和音〜テンションコードまで全対応、3和音と4和音の混在可）
- タップ → 同ボトムシートで編集
- 左スワイプで削除、長押しで並べ替え
- 作成したテンプレートはレイヤー設定の「コード進行」テンプレートドロップダウンにも反映

### Finder（識別）

指板をタップして音を選択し、その音の組み合わせに一致するコード/スケールを自動検出する機能（フッター2タブ目）。

#### コード識別

- **音名単位でトグル選択**: タップした音名が全弦まとめてハイライト表示される
- **ルート音設定**: 初期状態はルート未選択。指板を長押しでルート音を設定（extraNotes クリア）。ヘッダーの ‹ › はFinderタブでは非表示
- **完全一致**: 選択した音の集合がコードの構成音と完全に一致するコードを表示（構成音数の多い順）
- **含まれる**: コードの構成音が全て選択音の中に収まるコードを表示（構成音数の多い順）
- **含む**: 選択した音が全てコードに含まれるもの（選択音がコードのサブセット）を表示（構成音数の少ない順＝完成に近い順）
- **構成音表示**: 音名/度数はフレットボードのbaseLabelModeに連動して切り替わる
- **リセット**: 選択チップのタップ、またはリセットボタンでクリア
- **レイヤーに追加**: 結果一覧のコード/スケールをタップすると確認モーダルが表示される。確認後、ホームタブへ遷移してレイヤーとして追加される。レイヤーが3つ埋まっている場合はモーダルで追加不可の旨を通知する。追加時の色は自動で重複しない色が選ばれる

### クイズ機能

5種類のクイズモード:
- **音名クイズ**: 指板上の音名を当てる（12択 / 指板タップ）
- **度数クイズ**: 指板上の度数を当てる（12択 / 指板タップ）
- **コード識別**: コードフォームからルート音とコード種別を回答
- **スケールクイズ**: スケールの構成音を選択
- **ダイアトニック**: ダイアトニックコードの構成を回答

#### 苦手分析

クイズ画面内から開ける、回答履歴に基づいて苦手傾向を可視化するページ。

- **弦×フレット ヒートマップ**: 指板レイアウトと同じ向きで正答率を色で表示（赤=苦手 / 緑=得意）
- **モード別**: 音名・度数・コード・スケール・ダイアトニックの各クイズモードごとの正答率
- **音名別**: 全12音の正答率ランキング
- **度数別**: 全12度数の正答率ランキング
- **弦別**: 6弦それぞれの正答率ランキング
- **フレット別**: 0〜14フレットそれぞれの正答率ランキング
- **コード種別**: 全コードタイプの正答率ランキング
- **スケール別**: 全スケールタイプの正答率ランキング
- セクションは折りたたみ式、ヒートマップのみ常時展開
- データが5問未満のエントリは薄く表示（`--`）、0問は未解答として区別
- 記録はアプリを閉じても保持（AsyncStorage、最大1000件）
- リセットボタンで全記録を削除

### 設定

- テーマ: ダーク / ライト
- 臨時記号: ♯ / ♭
- 左利きモード: 指板を左右反転表示
- フレット範囲指定
- 言語: 日本語 / 英語

## UI デザイン（iOS 26）

iOS 26 のデザインパターンに準拠したUIを採用。

- **ヘッダータイトル**: 各タブのページタイトルを左大文字で表示（レイヤー / Finder / クイズ / 管理）
- **ルート音・表示切替**: ヘッダーから指板下へ移動。指板との関連性を視覚的に明確化
- **同心円角丸**: ボトムシート（28px）と内部要素の角丸を同心円ルールで統一
- **クイズスワイプ**: クイズパネルの左端に `borderTopLeftRadius: 28 / borderBottomLeftRadius: 28` を付与
- **GlassIconButton**: ×・‹・? などアイコン系ボタンを `expo-glass-effect` の `GlassView`（iOS Liquid Glass）で統一した共通コンポーネント（44pt 標準サイズ）
- **ハンドル余白**: 共通定数 `SHEET_HANDLE_CLEARANCE = 28` を `BottomSheetModal` からエクスポートし、全ボトムシートのコンテンツ上部余白を統一
- **Progressive Blur ヘッダー**: `SheetProgressiveHeader` 共通コンポーネントで iOS 26 風のフロストガラスヘッダーを全ボトムシートに適用。`expo-blur` の `BlurView` が全幅を覆い、`expo-linear-gradient` の `LinearGradient` がシート背景色へグラデーションでフェードする

- **Runtime**: Bun
- **Framework**: Expo + React Native
- **Language**: TypeScript (strict mode)
- **UI**: React 19
- **SVG**: react-native-svg
- **i18n**: i18next + react-i18next
- **Storage**: @react-native-async-storage/async-storage

bun start

bun run fmt
bun run lint
